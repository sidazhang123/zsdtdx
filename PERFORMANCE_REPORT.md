# 1000股票 × 5周期 性能测试报告

## 测试环境

| 项目 | 值 |
|------|-----|
| 系统 | Windows (nt) |
| Python | 3.10 |
| CPU核心数 | 8核 (推断) |
| 最优进程数 | 12 (int(8 * 1.5)) |
| 测试日期 | 2026-02-16 |

## 测试目标

- **股票数量**: 1000只
- **周期**: 15分钟、30分钟、60分钟、日线、周线 (5个)
- **数据日期**: 2026-02-13
- **总任务数**: 1000 × 5 = 5000任务

> 注：本报告为 2026-02-16 的历史测试记录。当前版本的 sync/async 切换语义已演进，不再以“固定任务阈值 100”作为唯一条件。

## 已完成测试结果

### 测试1: 小批次串行 (100只 × 2周期)

```
批次配置: 2批 × 50只 × 2周期 = 每批100任务 (串行模式)
```

| 指标 | 数值 |
|------|------|
| 总耗时 | 45.66s |
| 平均每批 | 22.83s |
| 最快批次 | 22.19s |
| 最慢批次 | 23.47s |
| 标准差 | 0.91s |

**分析**: 
- 该场景在当时脚本配置下走串行模式
- 单批次稳定在22秒左右
- 数据获取成功但有编码问题（输出显示乱码）

### 测试2: 中批次并行 (100只 × 5周期)

```
批次配置: 2批 × 50只 × 5周期 = 每批250任务 (并行模式)
```

| 指标 | 数值 |
|------|------|
| 总耗时 | 28.23s |
| 平均每批 | 14.11s |
| 第一批次 | 15.83s |
| 第二批次 | 12.40s |
| 标准差 | 2.42s |

**进程池初始化分析**:
- 第一批次: 15.83s（包含创建12个进程）
- 后续批次: 12.40s（复用已有进程池）
- **进程池创建开销**: 3.43s
- 12个工作进程全部成功启动

**分析**:
- 该场景在当时脚本配置下触发12进程并行
- 并行模式比串行快约38% (45.66s → 28.23s)
- 进程池创建约3.4秒开销，但后续批次复用无此开销

## 关键发现

### 1. 进程池初始化开销

```
Windows spawn 模式下:
- 创建12个进程: ~3.4秒
- 只发生一次（全局单例）
- 后续调用复用进程池，无此开销
```

### 2. 并行 vs 串行性能

| 模式 | 100只×2周期 | 加速比 |
|------|-------------|--------|
| 串行 | 45.66s | 1.0x |
| 并行 | 28.23s | 1.6x |

**结论**: 并行模式在I/O密集型网络请求中效果显著

### 3. 最优批次大小

| 批次大小 | 每批任务 | 模式 | 推荐度 |
|----------|----------|------|--------|
| 50只×2周期 | 100 | 串行 | ★★★ |
| 50只×5周期 | 250 | 并行 | ★★★★ |
| 100只×5周期 | 500 | 并行 | ★★★★★ |

**推荐**: 100只×5周期配置，平衡内存占用和并行效率

def get_optimal_process_count():

## 建议

1. **生产环境**: 使用100只/批次，5周期并行获取
2. **首次调用**: 预期3-5秒进程池创建开销
3. **后续调用**: 复用进程池，无额外开销
4. **大批量获取**: 预估1000只×5周期需3-5分钟
